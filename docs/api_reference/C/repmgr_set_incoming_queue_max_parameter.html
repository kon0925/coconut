repmgr_set_incoming_queue_max

Sets a byte-count limit on the amount of dynamic memory used by the Replication Manager incoming queue.

The syntax of this parameter in the DB_CONFIG file is a single line with the string repmgr_set_incoming_queue_max, one or more whitespace characters, and the limit specified in two parts: the gigabytes and the bytes values. For example: 

repmgr_set_incoming_queue_max 0 104857600

Sets a 100 megabyte limit.

For more information, see DB_ENV->repmgr_set_incoming_queue_max() .
